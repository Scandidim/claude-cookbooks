# SCANDIDIM: План впровадження 30/60/90

## 0–30 днів (Stabilize)

- Затвердити протокол і RACI.
- Перевести 100% нових задач у TASK_QUEUE.
- Запустити щоденний ритм (start/mid/end day).
- Налаштувати SLA контроль статусів (15 хв).
- Перший тижневий аудит: частка задач без DoD, частка невалідних ескалацій.

### Цільові метрики
- >= 90% задач мають owner + deadline + status.
- <= 10% задач без Definition of Done.
- <= 15% прострочених апдейтів статусу.

## 31–60 днів (Standardize)

- Уніфікувати Definition of Done по основних типах задач.
- Включити повноцінний risk register.
- Включити автонагадування щодо BLOCKED > 24h.
- Запустити щотижневий операційний review з підсумковим score.

### Цільові метрики
- >= 95% задач оновлюються в рамках SLA.
- <= 5% ескалацій у CEO без повного шаблону.
- >= 80% BLOCKED вирішуються < 48h.

## 61–90 днів (Scale)

- Запустити рольові дашборди (CEO / Lead / Owner).
- Впровадити автоматичну класифікацію ризиків і пріоритезацію.
- Формалізувати ретроспективу процесних інцидентів.
- Підготувати Protocol #2 (performance governance).

### Цільові метрики
- >= 98% задач відповідають стандарту даних.
- <= 3% задач “без руху” > 72h.
- >= 90% прогнозованість дедлайнів по пріоритетах P0/P1.
